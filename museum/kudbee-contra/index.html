How it was built

From idea to shipped

Four real beats — no simulated content in this section, every line traces to the game's own README and build history.

  1. The idea

    Kudbee Games Studio wanted a flagship title — something that proves the studio ships full, replayable products, not tech demos. A 2.5D run-and-gun was the pick: readable, physical, and hard to fake.

  2. Built by agents

    The engine, entities, and level data were authored through iterative Claude Code agent sessions — classic <script> files attached to one global namespace (window.KC), no bundler, no framework. Art and audio are procedural, generated in code rather than fetched from a runtime AI provider.

  3. Iteration

    Phase 1 shipped as a vertical slice: a full moveset for the Kudbee Operative and the K9 companion, seven parallax depth layers, pooled particles and projectiles, and a two-phase Hive Sentinel mini-boss — refined across build passes tracked in the studio's own ledger.

  4. Shipped

    Free, in your browser, right now — no install, no signup, no paywall. It runs from a static file server exactly the way it runs in production.
